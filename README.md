# Report Export Service

## Описание

Сервис предоставляет API для анализа текстовых файлов и построения частотной статистики по словоформам.

Эндпоинт:
`POST /public/report/export`  
Принимает текстовый файл и возвращает Excel-файл (xlsx), содержащий:

- словоформу (нормальную форму слова)
- общее количество вхождений в документе
- количество вхождений в каждой строке (в формате строки, например: `0,11,32,0,0,3`)

Поддерживаются большие файлы: обработка выполняется построчно без загрузки всего файла в память.

---

## Стек и принятые решения

- **FastAPI** — для реализации HTTP API
- **pymorphy3** — для лемматизации (нормализации словоформ)
- **openpyxl** — для генерации Excel-файлов

### Почему pymorphy3

Библиотека выбрана как лёгкое и эффективное решение для русского языка:

- не требует загрузки тяжёлых ML-моделей
- быстро работает “из коробки”
- обеспечивает достаточную точность для задачи частотного анализа

Альтернативы (например, spaCy) были бы избыточны для данного кейса.

## Архитектура и структура проекта

Проект организован по модульному принципу с разделением по бизнес-смыслу (inspired by DDD, но в упрощённом виде).

```text
app/
├── main.py
├── report/
│   ├── router.py        # HTTP-эндпоинт
│   ├── service.py       # основной сценарий (use case)
│   ├── analyzer.py      # анализ текста и сбор статистики
│   └── excel_exporter.py # генерация Excel-файла

tests/
└── report/
    ├── test_analyzer.py
    └── test_excel_exporter.py
```

## Запуск проекта
1. Клонировать репозиторий
```
git clone <repo_url>
cd <repo_name>
```
2. Создать виртуальное окружение  
```
python -m venv venv
```

3. Активировать виртуальное окружение:  
```
Windows:
venv\Scripts\activate

Linux / macOS:
source venv/bin/activate
```

3. Установить зависимости  
```
pip install -r requirements.txt
```
4. Запустить сервер
```
uvicorn app.main:app --reload
```
5. Открыть Swagger UI
```
http://127.0.0.1:8000/docs
```
